# Concepts overview

## Glossary: objective vs task loss

EmbedKD keeps two kinds of losses strictly apart:

- An **objective** (`embedkd.objectives`) aligns the student's embedding
  space with the teacher's: `cosine`, `mse`, `kl`, `rkd`.
- A **task loss** (`embedkd.losses`) trains the retrieval task itself on the
  student: `sce`, `arcface`, `triplet`, `contrastive`.

The total loss is `task + distill.alpha * objective`. Setting
`distill.alpha: 0` trains a model standalone (that is how teachers are made).

## Numerical safety

The backbone forward pass runs under autocast when AMP is on, but every loss
is computed in fp32 on upcast embeddings outside the autocast region.
Geometric losses (RKD, triplet) underflow in fp16: tiny pairwise distances
turn into infinite gradients, GradScaler silently skips steps, and training
freezes while looking alive. fp32 loss computation is the default for
everything, not an opt-in.

## Reproducibility model

- `train.seed` seeds python, numpy, torch, CUDA and DataLoader workers;
  cuDNN runs in deterministic mode.
- Same machine + same seed is bit-exact (an automated test enforces it).
- Every run writes `fingerprint.yaml`: the resolved config, seed, library
  versions, git commit and hardware. Published numbers are generated by
  committed scripts from these logs, never typed by hand.

## One code path

The CLI is a thin shell over `embedkd.DistillationRun`. Whatever you can do
with `embedkd fit`, you can do in a notebook:

```python
from embedkd import DistillationRun

run = DistillationRun.from_config("configs/quickstart_cpu.yaml",
                                  overrides=["train.epochs=1"])
result = run.fit()
print(run.evaluate(checkpoint=result["checkpoints"]["best"]))
```
